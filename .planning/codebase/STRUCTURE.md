# Codebase Structure

**Analysis Date:** 2026-07-02

## Directory Layout

```
[project-root]/
├── .agents/            # GSD agent config, skills, workflows, and templates
├── backend/            # Express.js REST API and Puppeteer Scraper
│   ├── prisma/         # Prisma schema, seed scripts, and SQL migrations
│   └── src/            # Backend TypeScript source code
│       ├── controllers/# Controller definitions (empty/optional placeholders)
│       ├── middleware/ # Express middleware (e.g., error handler)
│       ├── routes/     # REST API routes (display API and scraper triggers)
│       ├── services/   # Scrapers, RSS aggregators, and background jobs
│       ├── types/      # TypeScript declarations
│       └── utils/      # Prayer time and timezone offset calculators
├── frontend-display/   # Create React App dashboard UI
│   ├── src/            # React TypeScript source code
│   │   ├── assets/     # Branding assets, logos, and illustrations
│   │   ├── components/ # Individual dashboard widget cards
│   │   │   └── ui/     # Core primitives (radial backgrounds, pills)
│   │   ├── hooks/      # Custom React hooks for data aggregation
│   │   └── lib/        # Utility styling functions (e.g., tailwind merger)
│   └── nginx.conf      # Production Nginx reverse-proxy configuration

├── docker-compose.yml  # Orchestrates PostgreSQL, backend, and frontend containers
└── README.md           # Developer startup documentation
```

## Directory Purposes

**backend/src/routes:**
- Purpose: Exposes HTTP endpoints to the frontend.
- Contains: `display.ts` (retrieves dashboard schedules and ticker feeds) and `sync.ts` (triggers scraper run).
- Key files: `display.ts`.

**backend/src/services:**
- Purpose: Integrates with third-party sites and data streams.
- Contains: `scraper.ts` (crawls meetings using Puppeteer) and `ticker.ts` (caches BMKG, RSS feeds, CSV documents, and rates).
- Key files: `scraper.ts`.

**backend/prisma:**
- Purpose: Manages database models, tables, and seeding logic.
- Contains: `schema.prisma` (PostgreSQL schema definition) and `seed.ts` (initial dummy data).
- Key files: `schema.prisma`.

**frontend-display/src/components:**
- Purpose: Modular interface widgets optimized for TV viewports.
- Contains: Widgets such as `PrayerTimes.tsx`, `MeetingList.tsx`, `HourlyWeather.tsx`, and `AnnouncementTicker.tsx`.
- Key files: `MeetingList.tsx` (handles dynamic scrolling agenda lists).

**stitch:**
- Purpose: Acts as a design-reference sandbox. Contains HTML prototypes and visuals.
- Contains: Static pages mirroring product specifications. Not part of the runtime code.

## Key File Locations

**Entry Points:**
- `backend/src/index.ts`: Boots Express API, triggers initial scrape, starts cron schedules.
- `frontend-display/src/index.tsx`: Launches React UI.

**Configuration:**
- `docker-compose.yml`: Binds database, API, and static file servers.
- `backend/package.json` / `frontend-display/package.json`: Dependency manifests.
- `.env.example` / `.env`: Global environment blueprints.

**Core Logic:**
- `backend/src/services/scraper.ts`: Scraper engine.
- `backend/src/utils/prayerTimes.ts`: Local prayer time calculations.

**Testing:**
- `frontend-display/src/components/MeetingList.test.tsx`: Visual layout unit tests.

**Documentation:**
- `README.md`: Set up instructions.
- `BLACKBOX.md`: Layout design details.

## Naming Conventions

**Files:**
- `kebab-case.ts / .tsx`: Standard files in backend/frontend.
- `PascalCase.tsx`: React components (`MeetingList.tsx`, `CanvaEmbed.tsx`).
- `*.test.tsx`: Test runner specs.
- `camelCase.ts`: Utility files (`prayerTimes.ts`).

**Directories:**
- `kebab-case`: General directories (`frontend-display`).
- Plural nouns for folders: `components`, `routes`, `services`, `hooks`.

## Where to Add New Code

**New Scraper / Background Task:**
- Implementation: `backend/src/services/[task-name].ts`
- Scheduling: Wire into cron scheduler inside `backend/src/index.ts`.

**New Dashboard Widget:**
- Component code: `frontend-display/src/components/[WidgetName].tsx`
- Layout: Mount inside grid layout in `frontend-display/src/App.tsx`.
- Mocking/Testing: Create companion file `frontend-display/src/components/[WidgetName].test.tsx`.

**New Route Handler:**
- Definition: Add to `backend/src/routes/` under appropriate namespace.
- Register: Initialize in `backend/src/index.ts`.

## Special Directories

**backend/dist:**
- Purpose: Compiled Javascript output for backend production.
- Source: Generated by `npm run build` (`tsc`).
- Committed: No (in `.gitignore`).

**frontend-display/build:**
- Purpose: Production optimized assets for the frontend.
- Source: Generated by `npm run build` (`react-scripts build`).
- Committed: No (in `.gitignore`).
