# Requirements: PPKASN Corporate TV Display

**Defined:** 2026-07-02
**Core Value:** Continuous, reliable, and highly legible information broadcast on TV displays across different physical locations, manageable directly via a secure CMS admin panel (Single Source of Truth).

## v1 Requirements

Requirements for mobile responsiveness in Milestone v4.0.

### Agenda Display Responsiveness (Opsi 2)

- [x] **MOB-01**: User can view the public TV Agenda display routes (`/` and `/agenda`) on small viewports without horizontal scrolling.
- [x] **MOB-02**: Mobile display view hides secondary widgets (Video Embed and Weather) to prioritize screen real estate.
- [x] **MOB-03**: Mobile display view stacks the main widgets (Header, Agenda/Meeting List, Prayer Times, Announcement Ticker) vertically.

### CMS Admin Navigation

- [x] **MOB-04**: User can open and close the CMS Admin sidebar drawer on smartphone viewports using a hamburger menu toggle button.
- [x] **MOB-05**: Tapping the backdrop overlay outside the open sidebar drawer closes the sidebar automatically.
- [x] **MOB-06**: Selecting any sidebar navigation link closes the drawer to reveal the corresponding content panel.

### CMS Admin Forms & Data Tables

- [x] **MOB-07**: CMS Admin input forms (Agenda and Cuti) fit within smartphone viewports without horizontal overflow.
- [x] **MOB-08**: CMS Admin data tables (Agenda list and Cuti list) adjust their layout or allow horizontal scrolling to prevent content overflow.
- [x] **MOB-09**: Interactive buttons and inputs in CMS Admin have minimum tap targets of 44x44px on mobile devices.

## v2 Requirements

Deferred to future releases.

- **MOB-10**: Support for horizontal swipe gestures to open/close the CMS Admin drawer menu.
- **MOB-11**: Offline caching of display schedules on local smartphone storage.

## Out of Scope

Explicitly excluded.

| Feature | Reason |
|---------|--------|
| Multi-column grid on mobile | TV displays are designed for 1080p landscape; forcing multiple columns on a smartphone makes text unreadable. |
| Native App wrapping | We are building a responsive web layout, not compiling a native iOS/Android binary. |
