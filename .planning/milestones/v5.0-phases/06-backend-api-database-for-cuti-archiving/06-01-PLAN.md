## <objective>
This plan achieves the implementation of database schema modifications and backend API changes to support employee leave (cuti) archiving (soft-delete, restore, and filtering) as well as updating the public display client API to hide archived leaves.
By soft-deleting leaves (changing `isArchived` flag to true) rather than deleting database rows, we maintain historical leave records while allowing the CMS admin UI to organize and clean up active items and preventing past/archived items from appearing on TV displays.
